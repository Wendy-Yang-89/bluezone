# 渲染排序机制详解

## 背景与问题引入

### 渲染排序的重要性

**渲染排序** 决定场景中物体的绘制顺序，直接影响：

| 影响维度 | 正确排序的作用 |
|---------|---------------|
| **透明物体** | Back-to-Front排序确保混合正确 |
| **性能优化** | 前后排序减少overdraw |
| **渲染效果** | 特定层顺序保证效果正确 |
| **资源状态** | 正确状态切换避免渲染错误 |

错误排序可能导致透明物体穿透、状态污染、性能下降等问题。

### 三层排序机制

Lume3D采用三层排序参数控制渲染顺序：

| 参数 | 范围 | 作用 |
|------|------|------|
| `renderSortLayer` | 0-63 | 粗粒度层级排序 |
| `renderSortLayerOrder` | 0-255 | 层内细粒度排序 |
| `sortType` | 枚举值 | 层内深度排序方式 |

三层参数形成 `Layer → Order → Depth` 的排序链。

### 排序规则

排序规则的核心理念：

- **数值越小，渲染越早**（0最先，63/255最后）
- **Layer决定宏观顺序**（如UI在最前，背景在最后）
- **Order决定层内顺序**（如粒子A在粒子B前）
- **sortType决定深度方向**（Near-to-Far或Far-to-Near）

理解排序规则是正确配置物体渲染顺序的前提。

### 本文档解决的问题

本文档详细分析三层排序参数的定义和排序算法实现：

- 数据结构定义和默认值
- 各参数的作用和配置方式
- sortKey的生成算法和排序规则

## 核心概念

### renderSortLayer

**renderSortLayer** 是粗粒度渲染层级：

| 属性 | 说明 |
|------|------|
| **类型** | uint8_t |
| **范围** | 0-63 |
| **默认值** | 32（中间层） |
| **排序规则** | 小值先渲染 |

典型应用：
- Layer 0：UI层（最先渲染）
- Layer 32：普通物体（默认）
- Layer 63：背景层（最后渲染）

Layer实现渲染层级的宏观划分。

### renderSortLayerOrder

**renderSortLayerOrder** 是Layer内的细粒度顺序：

| 属性 | 说明 |
|------|------|
| **类型** | uint8_t |
| **范围** | 0-255 |
| **默认值** | 0 |
| **排序规则** | 小值先渲染 |

典型应用：
- Order 0：层内首个物体
- Order 128：层内中间物体
- Order 255：层内最后物体

Order实现同一Layer内的精细控制。

### sortType

**sortType** 决定Layer内的深度排序方式：

| 枚举值 | 排序方向 | 适用场景 |
|--------|---------|---------|
| `NONE` | 不排序 | 默认值 |
| `FRONT_TO_BACK` | 近→远 | 不透明物体优化 |
| `BACK_TO_FRONT` | 远→近 | 透明物体 |
| `BY_MATERIAL` | 按材质分组 | 不透明物体 |

sortType影响sortKey的深度部分计算方式。

### 三个排序关键值

排序涉及三个关键值，分为两个层级：

| 值 | 类型 | 来源 | 作用 |
|----|------|------|------|
| **sortLayerKey** | uint16_t → uint32_t | `combinedRenderSortLayer` | 宏观层级排序（第一优先级） |
| **renderSortHash** | uint32_t | `materialIndex ⊕ renderHash` | 材质/状态分组（sortKey组成部分） |
| **sortKey** | uint64_t | `renderSortHash` + `depth` | 层内精细排序 |

#### 1. renderSortLayerHash（→ sortLayerKey）

`GetRenderSortLayerHash` 将 Layer + Order 合并为 16 位整数，存入 `SlotMaterialData.combinedRenderSortLayer`，
最终成为 `SlotSubmeshIndex.sortLayerKey`：

```
renderSortLayerHash = (renderSortLayer << 8) | (renderSortLayerOrder & 0xFF)
```

若 mesh 值为默认值(32,0)，则使用 material 值；否则使用 mesh 值。

示例：Layer=10, Order=5 → 0x0A05 = 2565

#### 2. renderSortHash

`renderSortHash` 是 32 位哈希值，存入 `SlotMaterialData.renderSortHash`，用于 sortKey 的材质/状态分组部分：

```
renderSortHash = materialIndex
HashCombine32Bit(renderSortHash, renderHash)

其中 HashCombine32Bit:
  seed ^= v + 0x9e3779b9 + (seed << 6) + (seed >> 2)

renderHash = HashSubmeshMaterials(materialType, renderMaterialFlags, submeshFlags)
```

#### 3. sortKey

**sortKey** 的位分布由 sortType 决定：

```
BY_MATERIAL模式:
┌──────────────────────────┬──────────────────────────┐
│   高32位: renderSortHash │   低32位: depth           │
└──────────────────────────┴──────────────────────────┘
→ renderSortHash 主导排序，相同材质聚组后按深度排列

FRONT_TO_BACK / BACK_TO_FRONT模式:
┌────────────────────────┬───────────────────────────┐
│   高32位: depth        │   低32位: renderSortHash   │
└────────────────────────┴───────────────────────────┘
→ depth 主导排序，相同深度内按材质排列
```

### 排序流程

```
排序流程:

┌──────────────────────────────────────────────────┐
│ 输入参数                                          │
├──────────────────────────────────────────────────┤
│ renderSortLayer (0-63)                           │
│ renderSortLayerOrder (0-255)                     │
│ materialIndex, materialType, renderMaterialFlags │
│ submeshFlags                                     │
│ sortType (枚举)                                  │
│ depth (相机距离)                                  │
└──────────────────────────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────┐
│ 1. 计算 sortLayerKey                             │
├──────────────────────────────────────────────────┤
│ mesh值为默认(32,0)时取material值，否则取mesh值     │
│ sortLayerKey = (layer << 8) | (order & 0xFF)     │
│ 例: Layer=32, Order=0 → 0x2000                   │
└──────────────────────────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────┐
│ 2. 计算 renderSortHash                           │
├──────────────────────────────────────────────────┤
│ renderHash = HashSubmeshMaterials(...)           │
│ renderSortHash = materialIndex                   │
│ renderSortHash ^= renderHash + goldenRatio       │
│                  + (renderSortHash << 6)         │
│                  + (renderSortHash >> 2)         │
└──────────────────────────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────┐
│ 3. 计算 sortKey (根据sortType)                    │
├──────────────────────────────────────────────────┤
│ depth = min(maxDepth, abs(z) * coefficient)      │
│                                                  │
│ BY_MATERIAL:                                     │
│   sortKey = (renderSortHash << 32) | depth       │
│ FRONT_TO_BACK / BACK_TO_FRONT:                   │
│   sortKey = (depth << 32) | renderSortHash       │
└──────────────────────────────────────────────────┘
                      │
                      ▼
┌───────────────────────────────────────────────────┐
│ 4. 最终排序                                        │
├───────────────────────────────────────────────────┤
│ 1. 按sortLayerKey升序（值小先渲染）                 │
│ 2. 同一sortLayerKey内按sortKey排序                 │
│    - Less: sortKey升序（BY_MATERIAL/FRONT_TO_BACK）│
│    - Greater: sortKey降序（BACK_TO_FRONT）         │
└───────────────────────────────────────────────────┘
```

## 三、排序优先级体系

渲染系统的多级排序优先级：

```
1. RenderSlot（渲染槽） ── 最高优先级，决定渲染管线
   │
2. sortLayerKey（combinedRenderSortLayer） ── 层级键升序
   │
3. sortKey ── 由sortType决定计算方式：
   ├── BY_MATERIAL: (renderSortHash << 32) | depth
   │   → renderSortHash升序 + depth升序
   ├── FRONT_TO_BACK: (depth << 32) | renderSortHash
   │   → depth升序（近→远）
   └── BACK_TO_FRONT: (depth << 32) | renderSortHash
       → depth降序（远→近）

其中:
  sortLayerKey = (layer << 8) | (order & 0xFF)    ← 16位层级哈希
  renderSortHash = materialIndex ⊕ renderHash      ← 32位材质/状态哈希
```

## 四、排序算法实现

### 4.1 renderSortLayerHash 计算（→ sortLayerKey）

**文件位置：** `src/render/datastore/render_data_store_default_material.cpp:198-209`

```cpp
inline constexpr uint16_t GetRenderSortLayerHash(const RenderSubmesh& submesh)
{
    // if submesh layers are defaults -> use material sort values (the defaults are the same)
    if ((submesh.layers.meshRenderSortLayer == RenderSceneDataConstants::DEFAULT_RENDER_SORT_LAYER_ID) &&
        (submesh.layers.meshRenderSortLayerOrder == 0)) {
        return (static_cast<uint16_t>(submesh.layers.materialRenderSortLayer) << 8u) |
               (static_cast<uint16_t>(submesh.layers.materialRenderSortLayerOrder) & 0xffu);
    } else {
        return (static_cast<uint16_t>(submesh.layers.meshRenderSortLayer) << 8u) |
               (static_cast<uint16_t>(submesh.layers.meshRenderSortLayerOrder) & 0xffu);
    }
}
```

**计算公式：**
```
renderSortLayerHash = (renderSortLayer << 8) | (renderSortLayerOrder & 0xFF)
```

此值存入 `SlotMaterialData.combinedRenderSortLayer`，最终成为 `SlotSubmeshIndex.sortLayerKey`。

**示例：**
- renderSortLayer = 10, renderSortLayerOrder = 5 → Hash = 0x0A05 = 2565
- renderSortLayer = 20, renderSortLayerOrder = 3 → Hash = 0x1403 = 5123

---

### 4.2 renderSortHash 计算

**文件位置：** `src/render/datastore/render_data_store_default_material.cpp:1643-1660`

`renderSortHash` 在 `AddSubmeshToRenderSlots` 中计算，是 materialIndex 与 renderHash 的组合哈希：

```cpp
uint32_t renderSortHash = submesh.indices.materialIndex;
HashCombine32Bit(renderSortHash, renderHash);
```

**HashCombine32Bit** (`render_data_store_default_material.cpp:109-115`)：

```cpp
inline void HashCombine32Bit(uint32_t& seed, const uint32_t v)
{
    constexpr const uint32_t goldenRatio = 0x9e3779b9;
    constexpr const uint32_t rotl = 6U;
    constexpr const uint32_t rotr = 2U;
    seed ^= uint32_t(v) + goldenRatio + (seed << rotl) + (seed >> rotr);
}
```

**renderHash** 由 `HashSubmeshMaterials` 生成，基于 `materialType`、`renderMaterialFlags`、`submeshFlags` 计算。

**公式总结：**
```
renderHash = HashSubmeshMaterials(materialType, renderMaterialFlags, submeshFlags)
renderSortHash = materialIndex
renderSortHash ^= renderHash + 0x9e3779b9 + (renderSortHash << 6) + (renderSortHash >> 2)
```

此值存入 `SlotMaterialData.renderSortHash`，参与 sortKey 的材质/状态分组部分。

---

### 4.3 Submesh vs Material 优先级

```
优先级规则：
1. Submesh 的排序值优先于 Material 的排序值
   - 如果 submesh.renderSortLayer != 32（默认值）
   - 则使用 submesh 的排序值

2. 否则使用 Material 的排序值
   - 如果 submesh 使用默认值
   - 则继承 material 的排序值
```

### 4.4 sortKey 计算（64位）

**文件位置：** `src/render/render_node_scene_util.cpp:337-377`

```cpp
constexpr uint64_t maxUDepth = RenderDataStoreDefaultMaterial::SLOT_SORT_MAX_DEPTH;
constexpr uint64_t sDepthShift = RenderDataStoreDefaultMaterial::SLOT_SORT_DEPTH_SHIFT;
constexpr uint64_t sRenderMask = RenderDataStoreDefaultMaterial::SLOT_SORT_HASH_MASK;
constexpr uint64_t sRenderShift = RenderDataStoreDefaultMaterial::SLOT_SORT_HASH_SHIFT;

// ...

const Math::Vec4 pos = (camView * Math::Vec4(submesh.bounds.worldCenter, 1.0f));
const float zVal = Math::abs(pos.z);
uint64_t sortKey = Math::min(maxUDepth, static_cast<uint64_t>(double(zVal) * camSortCoefficient));
if (renderSlotInfo.sortType == RenderSlotSortType::BY_MATERIAL) {
    // High 32bits for render sort hash, Low 32bits for depth
    sortKey |= (((uint64_t)submeshMatData.renderSortHash & sRenderMask) << sRenderShift);
} else {
    // High 32bits for depth, Low 32bits for render sort hash
    sortKey = (sortKey << sDepthShift) | ((uint64_t)slotSubmeshMatData[idx].renderSortHash & sRenderMask);
}
```

## 五、排序比较运算符

**文件位置：** `src/render/render_node_scene_util.cpp:50-72`

```cpp
inline bool operator<(const SlotSubmeshIndex& lhs, const SlotSubmeshIndex& rhs)
{
    // Sorting is always ascending by sortLayerKey. For equal sortLayerKeys sortKey (material/ depth) defines the
    // order.
    if (lhs.sortLayerKey < rhs.sortLayerKey) {
        return true;
    } else if (lhs.sortLayerKey == rhs.sortLayerKey) {
        return (lhs.sortKey < rhs.sortKey);
    }
    return false;
}

inline bool operator>(const SlotSubmeshIndex& lhs, const SlotSubmeshIndex& rhs)
{
    // Sorting is always ascending by sortLayerKey. For equal sortLayerKeys sortKey (material/ depth) defines the
    // order.
    if (lhs.sortLayerKey < rhs.sortLayerKey) {
        return true;
    } else if (lhs.sortLayerKey == rhs.sortLayerKey) {
        return (lhs.sortKey > rhs.sortKey);
    }
    return false;
}
```

**关键点：**
- `operator<`: sortLayerKey 升序 + sortKey 升序
- `operator>`: sortLayerKey 升序 + sortKey 降序
- sortLayerKey 总是升序（值越小越靠前）

## 六、不同 sortType 的排序规则详解

### 6.1 BY_MATERIAL 模式

**排序函数调用：**
```cpp
// render_node_scene_util.cpp:390
std::sort(refSubmeshIndices.begin(), refSubmeshIndices.end(), Less<SlotSubmeshIndex>());
```

**sortKey 位分布：**
```
[63 ... 32][31 ... 0]
 渲染排序Hash   深度值
```

**排序规则（使用 operator< 升序）：**

| 参数 | 排序方向 | 说明 |
|------|----------|------|
| **renderSortLayer** | 值越小越靠前 | 0最先渲染，63最后 |
| **renderSortLayerOrder** | 值越小越靠前 | 0最先渲染，255最后 |
| **depth（深度）** | 值越小越靠前（近→远） | 相同材质内从近到远渲染 |

**排序流程：**
```
1. 先按 sortLayerKey 升序（renderSortLayer 小到大）
2. 再按 sortKey 升序
   - sortKey 高32位 = renderSortHash（小到大）
   - sortKey 低32位 = depth（小到大）
3. 按材质分组，每组内找最小深度
4. 最终按 sortLayerKey + minDepth 升序排列
```

**适用场景：**
- 不透明物体渲染
- 减少状态切换（材质分组）
- 减少 overdraw（近→远渲染，深度测试剔除）

---

### 6.2 FRONT_TO_BACK 模式

**排序函数调用：**
```cpp
// render_node_scene_util.cpp:448
// front-to-back render layer sort is 0 -> 63
std::sort(refSubmeshIndices.begin(), refSubmeshIndices.end(), Less<SlotSubmeshIndex>());
```

**sortKey 位分布：**
```
[63 ... 32][31 ... 0]
   深度值      渲染排序Hash
```

**排序规则（使用 operator< 升序）：**

| 参数 | 排序方向 | 说明 |
|------|----------|------|
| **renderSortLayer** | 值越小越靠前 | 0最先渲染，63最后 |
| **renderSortLayerOrder** | 值越小越靠前 | 0最先渲染，255最后 |
| **depth（深度）** | 值越小越靠前（近→远） | 从前景到背景渲染 |

**渲染顺序图示：**
```
相机位置
    ↓
    ├─ depth=0（最近） → 最先渲染
    ├─ depth=5
    ├─ depth=10
    └─ depth=20（最远） → 最后渲染

方向：近 → 远（FRONT_TO_BACK）
```

**适用场景：**
- 不透明物体渲染（从近到远，减少overdraw）
- Early-Z优化
- 减少像素填充率

---

### 6.3 BACK_TO_FRONT 模式

**排序函数调用：**
```cpp
// render_node_scene_util.cpp:451
// back-to-front render layer sort is 63 -> 0
std::sort(refSubmeshIndices.begin(), refSubmeshIndices.end(), Greater<SlotSubmeshIndex>());
```

**sortKey 位分布：**
```
[63 ... 32][31 ... 0]
   深度值      渲染排序Hash
```

**排序规则（使用 operator> 降序）：**

| 参数 | 排序方向 | 说明 |
|------|----------|------|
| **renderSortLayer** | 值越小越靠前 | 仍为升序（0最先） |
| **renderSortLayerOrder** | 值越小越靠前 | 仍为升序（0最先） |
| **depth（深度）** | 值越大越靠前（远→近） | 从背景到前景渲染 |

**渲染顺序图示：**
```
相机位置
    ↓
    ├─ depth=20（最远） → 最先渲染
    ├─ depth=10
    ├─ depth=5
    └─ depth=0（最近） → 最后渲染

方向：远 → 近（BACK_TO_FRONT）
```

**适用场景：**
- 透明物体渲染（从远到近）
- 标准透明混合顺序
- 确保后渲染的透明物体正确覆盖

---

### 6.4 排序规则总结表

| sortType | renderSortLayer | renderSortLayerOrder | depth | 渲染方向 |
|-----------|-----------------|----------------------|-------|----------|
| **BY_MATERIAL** | 小→大（升序） | 小→大（升序） | 小→大（近→远） | 材质优先，减少状态切换 |
| **FRONT_TO_BACK** | 小→大（升序） | 小→大（升序） | 小→大（近→远） | 从前景到背景 |
| **BACK_TO_FRONT** | 小→大（升序） | 小→大（升序） | 大→小（远→近） | 从背景到前景 |

**关键结论：**
- `renderSortLayer` 和 `renderSortLayerOrder` 在所有模式下都是**值越小越靠前渲染**
- `depth` 的排序方向由 `sortType` 决定：
  - BY_MATERIAL / FRONT_TO_BACK: **值越小越靠前**（近→远）
  - BACK_TO_FRONT: **值越大越靠前**（远→近）

## 八、推荐的层级分配

| 层级范围 | 用途 | 示例 |

## 九、性能优化建议

### 9.1 BY_MATERIAL 模式优化

1. **减少状态切换**
   - 相同材质物体连续渲染
   - 避免频繁切换 shader、材质参数

2. **减少 overdraw**
   - 相同材质内按深度排序
   - 深度测试剔除被遮挡物体

3. **提高缓存命中率**
   - 材质数据局部性好
   - 纹理缓存更有效

---

### 9.2 FRONT_TO_BACK 模式优化

1. **正确的透明混合**
   - 深度大的物体先渲染
   - 正确的深度顺序

2. **材质分组**
   - 相同深度内按材质分组
   - 减少状态切换

---

### 9.3 视锥剔除优化

```cpp
// 优先渲染可能遮挡大面积的对象
largeObject->renderSort.renderSortLayer = 0;
```

## 十、常见问题

### Q1：为什么默认值是32？

**A：** 32是中间值，允许在前后插入其他层级，提供灵活性。

---

### Q2：Submesh 和 Material 的排序值如何交互？

**A：** Submesh 排序值优先，只有 Submesh 使用默认值(32, 0)时才使用 Material 的值。

---

### Q3：renderSortLayerOrder 范围为何是0-255？

**A：** 8位无符号整数，在排序Hash中占用低8位，提供足够精细度。
